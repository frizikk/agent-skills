## Overview

This Skill provides comprehensive support for interacting with the Google Jules AI coding assistant API. Jules is an asynchronous AI coding assistant that can help with code changes, PR creation, and development tasks.

## When to Use

Use this Skill when you need to:
- Create and manage Jules coding sessions
- Send prompts/tasks to Jules for processing
- Monitor session activities and progress
- Approve plans generated by Jules
- Work with GitHub repositories through Jules
- List and manage data sources

## Prerequisites

### Authentication

The Jules API uses Google Cloud authentication. You need to configure one of:

1. **OAuth 2.0** - For user authentication:
   ```bash
   gcloud auth application-default login
   ```

2. **Service Account** - For server-to-server authentication:
   ```bash
   export GOOGLE_APPLICATION_CREDENTIALS="/path/to/service-account.json"
   ```

### API Access

- The Jules API is available at: `https://jules.googleapis.com`
- API Version: `v1alpha`
- Enable the Jules API in Google Cloud Console

## Core Concepts

### Sessions

A **session** is a contiguous amount of work within the same context. Think of it as a single task or conversation with Jules.

**Session States:**
- `QUEUED` - Waiting to start
- `PLANNING` - Agent is planning the approach
- `AWAITING_PLAN_APPROVAL` - Waiting for user to approve the plan
- `AWAITING_USER_FEEDBACK` - Waiting for user input
- `IN_PROGRESS` - Actively working
- `PAUSED` - Temporarily paused
- `FAILED` - Failed to complete
- `COMPLETED` - Successfully completed

### Activities

An **activity** is a single unit of work within a session. Activities track:
- Agent messages
- User messages
- Plan generation and approval
- Progress updates
- Session completion/failure

### Sources

A **source** is an input data source for a session. Currently supported:
- GitHub repositories

## Working with Sessions

### Creating a Session

To create a new Jules session, you need:

1. A **prompt** - What you want Jules to do
2. A **source context** - Which GitHub repository to work with
3. Optional: **title**, **requirePlanApproval**, **automationMode**

**Example - Create a session to fix a bug:**

```python
import requests
import json

# First, get the source (GitHub repo)
sources_response = requests.get(
    "https://jules.googleapis.com/v1alpha/sources",
    headers={"Authorization": f"Bearer {access_token}"}
)
sources = sources_response.json()

# Find your repo source
repo_source = None
for s in sources.get("sources", []):
    if s.get("githubRepo", {}).get("owner") == "myorg" and s.get("githubRepo", {}).get("repo") == "myrepo":
        repo_source = s["name"]
        break

# Create session
session_data = {
    "prompt": "Fix the authentication bug in the login function where users with expired tokens are not properly redirected.",
    "sourceContext": {
        "source": repo_source,
        "githubRepoContext": {
            "startingBranch": "main"
        }
    },
    "title": "Fix authentication bug",
    "requirePlanApproval": True,
    "automationMode": "AUTO_CREATE_PR"  # Auto-create PR when done
}

response = requests.post(
    "https://jules.googleapis.com/v1alpha/sessions",
    headers={"Authorization": f"Bearer {access_token}"},
    json=session_data
)
session = response.json()
print(f"Session created: {session['name']}")
print(f"View at: {session['url']}")
```

### Sending Messages to a Session

After creating a session, you can send follow-up messages:

```python
session_name = "sessions/abc123"
message_data = {
    "message": "Also update the unit tests to cover this edge case."
}

response = requests.post(
    f"https://jules.googleapis.com/v1alpha/{session_name}:sendMessage",
    headers={"Authorization": f"Bearer {access_token}"},
    json=message_data
)
```

### Approving a Plan

When a session is in `AWAITING_PLAN_APPROVAL` state, you can approve the plan:

```python
session_name = "sessions/abc123"
approval_data = {

    f"https://jules.googleapis.com/v1alpha/{session_name}:approvePlan",
    headers={"Authorization": f"Bearer {access_token}"},
    json=approval_data
```

## Common Workflows

### Workflow 1: Full Bug Fix with PR

```python
def create_bug_fix_session(access_token, owner, repo, branch, prompt):
    """Create a session to fix a bug and auto-create a PR."""
    
    # Find the source
    sources_response = requests.get(
        "https://jules.googleapis.com/v1alpha/sources",
        headers={"Authorization": f"Bearer {access_token}"}
    )
    
    source_name = None
    for s in sources_response.json().get("sources", []):
        gh = s.get("githubRepo", {})
        if gh.get("owner") == owner and gh.get("repo") == repo:
            source_name = s["name"]
            break
    
    if not source_name:
        raise ValueError(f"Source not found for {owner}/{repo}")
    
    # Create session
    session_data = {
        "prompt": prompt,
        "sourceContext": {
            "source": source_name,
            "githubRepoContext": {
                "startingBranch": branch
            }
        },
        "requirePlanApproval": True,
        "automationMode": "AUTO_CREATE_PR"
    }
    
    response = requests.post(
        "https://jules.googleapis.com/v1alpha/sessions",
        headers={"Authorization": f"Bearer {access_token}"},
        json=session_data
    )
    
    return response.json()

# Usage
session = create_bug_fix_session(
    access_token="your-token",
    owner="myorg",
    repo="myrepo",
    branch="main",
    prompt="Fix the null pointer exception in user_service.py when handling empty user data"
)
print(f"Session: {session['name']}")
print(f"URL: {session['url']}")
```

### Workflow 2: Monitor Session Progress

```python
def wait_for_session_completion(access_token, session_name, poll_interval=10):
    """Poll session until completed, failed, or awaiting approval."""
    
    while True:
        response = requests.get(
            f"https://jules.googleapis.com/v1alpha/{session_name}",
            headers={"Authorization": f"Bearer {access_token}"}
        )
        session = response.json()
        state = session["state"]
        
        print(f"Session state: {state}")
        
        if state in ["COMPLETED", "FAILED"]:
            return session
        
        if state == "AWAITING_PLAN_APPROVAL":
            # Get the plan to show to user
            activities_response = requests.get(
                f"https://jules.googleapis.com/v1alpha/{session_name}/activities",
                headers={"Authorization": f"Bearer {access_token}"}
            )
            activities = activities_response.json().get("activities", [])
            
            # Find latest plan
            for activity in reversed(activities):
                if "planGenerated" in activity:
                    plan = activity["planGenerated"]["plan"]
                    print("\nProposed Plan:")
                    for step in plan["steps"]:
                        print(f"  {step['index']+1}. {step['title']}: {step['description']}")
                    return session
        
        time.sleep(poll_interval)
```

```python
def approve_plan(access_token, session_name, plan_id):
    """Approve a plan to allow Jules to proceed."""
    approval_data = {"planId": plan_id}
```

## Error Handling

```python
def jules_api_call(func, *args, **kwargs):
    """Wrapper for Jules API calls with error handling."""
    try:
        response = func(*args, **kwargs)
        
        if response.status_code == 401:
            raise Exception("Authentication failed. Check your access token.")
        elif response.status_code == 403:
            raise Exception("Permission denied. Check API permissions.")
        elif response.status_code == 404:
            raise Exception("Resource not found.")
        elif response.status_code >= 400:
            raise Exception(f"API error: {response.status_code} - {response.text}")
        
        return response.json()
    
    except requests.exceptions.RequestException as e:
        raise Exception(f"Request failed: {str(e)}")

# Usage
session = jules_api_call(
    requests.post,
    "https://jules.googleapis.com/v1alpha/sessions",
    headers={"Authorization": f"Bearer {access_token}"},
    json=session_data
)
```

## Best Practices

1. **Always require plan approval** for production code changes to review before execution
2. **Use automation mode** (`AUTO_CREATE_PR`) for automated workflows
3. **Monitor activities** to track progress and see agent messages
4. **Check session state** before sending messages or approving plans
5. **Handle rate limiting** - add retry logic with exponential backoff
6. **Store session URLs** to easily access the web UI for debugging
